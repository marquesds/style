# Skill Review Notes

Record skill edits that were proposed and rejected during review. The goal is to prevent the same change from being re-attempted, and to keep the reasoning visible to future contributors and agents.

This file is **not installed** into downstream agents. It is review knowledge for this repo only.

## Entry format

```
## <skill-id> — <short title of attempted change>

- **Date:** YYYY-MM-DD
- **PR:** <link or number>
- **Proposed:** what the edit was (1–3 sentences)
- **Rejected because:** the specific failure mode or trade-off (be concrete)
- **What to try instead:** a smaller bounded edit, an ADR, or "leave as-is"
```

Keep entries terse. One paragraph per field is plenty. Link to the PR for full context.
